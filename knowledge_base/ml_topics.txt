Machine Learning Fundamentals

Supervised Learning

Supervised learning is the most common ML paradigm, where a model is trained on labeled examples — pairs of input features and known output labels. The model learns a mapping from inputs to outputs that generalizes to unseen data. Common supervised algorithms include linear regression, logistic regression, decision trees, random forests, gradient boosting (XGBoost, LightGBM), support vector machines, and neural networks. Model quality is measured using metrics such as accuracy, F1 score, RMSE, and AUC-ROC.

Unsupervised Learning

Unsupervised learning discovers hidden patterns in data without labeled outputs. Key techniques include clustering (K-Means, DBSCAN, hierarchical clustering), dimensionality reduction (PCA, t-SNE, UMAP), and anomaly detection. These methods are used for customer segmentation, topic modeling, data compression, and exploratory data analysis.

Natural Language Processing (NLP)

Natural Language Processing (NLP) is the subfield of ML concerned with enabling computers to understand and generate human language. Core NLP tasks include text classification, named entity recognition (NER), sentiment analysis, machine translation, question answering, summarization, and dialogue systems. Modern NLP is dominated by transformer-based models pretrained on large text corpora.

Feature Engineering

Feature engineering is the process of selecting, transforming, and creating input variables (features) that improve model performance. Good features encode domain knowledge and reduce the learning burden on the model. Common techniques include normalization and standardization, one-hot encoding for categorical variables, TF-IDF for text, log transforms for skewed distributions, and creating interaction features. Feature engineering is often more impactful than model selection.

TF-IDF (Term Frequency-Inverse Document Frequency)

TF-IDF is a classical text feature extraction method that converts documents into numerical vectors. Term Frequency (TF) measures how often a word appears in a document. Inverse Document Frequency (IDF) penalizes words that appear in many documents (common words like "the"). The product TF × IDF produces high scores for words that are frequent in a specific document but rare across the corpus, making them good discriminating features. TF-IDF is fast, interpretable, and still competitive for short-text classification tasks.

Model Evaluation Metrics

Accuracy measures the fraction of correct predictions. Precision is the fraction of positive predictions that are actually positive. Recall (sensitivity) is the fraction of actual positives that are correctly identified. F1 score is the harmonic mean of precision and recall, useful when class distribution is imbalanced. AUC-ROC measures the model's ability to discriminate between classes at various thresholds. For regression, common metrics are Mean Absolute Error (MAE), Root Mean Squared Error (RMSE), and R-squared.

Overfitting and Regularization

Overfitting occurs when a model learns the training data too well, including noise, and fails to generalize to new data. Signs include high training accuracy but low test accuracy. Regularization techniques add penalties to model complexity: L1 regularization (Lasso) drives feature weights to zero, enabling feature selection; L2 regularization (Ridge) shrinks weights smoothly. Dropout, early stopping, and cross-validation are other key anti-overfitting strategies.

ML Pipelines and MLOps

An ML pipeline is a sequence of steps that transforms raw data into a deployed model. Steps typically include data ingestion, preprocessing, feature extraction, model training, evaluation, and deployment. MLOps (Machine Learning Operations) extends DevOps principles to ML systems, covering reproducibility, versioning (data, code, models), CI/CD for models, monitoring, and retraining automation. Tools include MLflow, DVC, Kubeflow, and SageMaker Pipelines.

Cloud AI Services

Major cloud providers offer managed AI/ML services that abstract infrastructure management. AWS services include SageMaker (end-to-end ML platform), Bedrock (LLM APIs), Rekognition (vision), and Comprehend (NLP). Azure offers Azure Machine Learning, Azure OpenAI Service, and Cognitive Services. Google Cloud provides Vertex AI, AutoML, and Document AI. Cloud AI services enable rapid prototyping and scalable deployment without managing underlying infrastructure.

Model Deployment and Serving

Model deployment is the process of making a trained model available for real-time or batch inference. Common patterns include: REST API (FastAPI or Flask wrapping model.predict()), batch pipelines (offline scoring of large datasets), streaming inference (real-time event processing), and serverless endpoints. Production concerns include latency, throughput, model versioning, A/B testing, canary deployments, and monitoring for data drift and model degradation.

Scikit-learn

Scikit-learn is the primary Python library for classical machine learning. It provides consistent API patterns (fit, predict, transform) across all algorithms, making model swapping easy. Key modules include sklearn.preprocessing (scaling, encoding), sklearn.feature_extraction (TF-IDF, count vectorizer), sklearn.linear_model, sklearn.ensemble, sklearn.svm, sklearn.metrics, and sklearn.pipeline. The Pipeline class is especially powerful as it chains preprocessing and modeling steps into a single, safe workflow that prevents data leakage.
